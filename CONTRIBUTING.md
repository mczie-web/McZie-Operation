# Contributing

## Flujo
1. Crear rama desde `main`.
2. Commits pequeños con mensaje claro.
3. Abrir PR con contexto: problema, solución, impacto.
4. No merge sin revisión.

## Convención de commits
- `feat:` nueva capacidad
- `fix:` corrección
- `chore:` mantenimiento
- `docs:` documentación

## Checklist de PR
- [ ] Cambios explicados en lenguaje de negocio
- [ ] Riesgos identificados
- [ ] Plan de validación definido
